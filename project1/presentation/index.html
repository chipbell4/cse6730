Project 1

Chip Bell

Basic Goals

  • Explore Event-Driven Simulation Design
  • Learn some CoffeeScript
  • Glean information from the NGSIM Data Set
  • Attempt to accurately simulate traffic at 12th and Peachtree

Basic Architecture Design

  • Written in CoffeeScript
  • Backbone was used for architecture and eventing
  • Fairly decoupled, but relies on both scheduled and unscheduled events
  • Models you'd expect: Car, EventQueue, IntersectionQueue
  • Views you wouldn't expect: StatView, LightTimingView, AnimationView
  • Real-time adjustment of light timing

Input Data

  • NGSIM Data Set
  • Imported into MySQL + Analyzed with JS
  • Result: Basic generalities (car length)
  • Result Input distribution hard to model. Used empirical distribution for simulation

Output Data (What do we want to learn?)

  • What does the output distribution look like?
  • How does light timing affect wait times?

Results

  • Output distribution is typically bimodal
  • Full Saturation occurs around 60:25 red/green ratio
